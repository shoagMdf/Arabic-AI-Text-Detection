# Data Directory

This directory contains all data artifacts produced by the pipeline, demonstrating the end-to-end data transformation from raw HuggingFace Parquet files through distributed processing to final predictions.

## Directory Structure

| Path | Format | Size | Description |
|------|--------|------|-------------|
| `raw/` | Snappy-Parquet | 22 MB | Original 8,388 Arabic abstracts from KFUPM-JRCAI HuggingFace dataset |
| `processed/` | Snappy-Parquet (4 partitions) | 49 MB | 36,524 samples after Arabic preprocessing (Hamza/Alif/Taa normalization, Tashkeel stripping, ISRI stemming) |
| `mapreduce_output.tsv` | Tab-separated text | 26 MB | Output of Hadoop Streaming MapReduce job: (token, count) pairs |
| `stream_input/` | JSON Lines | 404 KB | Simulated streaming input: 99 Arabic abstract JSON messages |
| `stream_output/` | Snappy-Parquet (12 batches) | 200 KB | Real-time predictions from Spark Structured Streaming |

## Reproducibility

All Spark jobs use fixed random seeds and deterministic preprocessing. To regenerate from scratch:

```bash
# Phase 1+2: Re-download from HuggingFace and preprocess
spark-submit src/data_preparation.py --hdfs

# Phase 3.1+3.2: Feature engineering
spark-submit src/feature_engineering.py \
    --in-dir hdfs://localhost:9000/user/$USER/arabic_nlp/data/processed \
    --out-dir data/features

# MapReduce job
python3 scripts/run_local_mapreduce.py

# Phase 4: Streaming
python3 src/streaming_pipeline.py produce --n-messages 99
spark-submit src/streaming_pipeline.py run --run-seconds 60
```

## Big Data Format Justification

- **Parquet** (columnar, Snappy-compressed): chosen for `raw/`, `processed/`, and streaming outputs because (a) it enables column pruning and predicate pushdown for fast Spark queries, (b) Snappy provides excellent compression ratios (~3-5×) with low CPU cost, and (c) it preserves schema across language boundaries (Python ↔ JVM).
- **JSON Lines** (newline-delimited): used for `stream_input/` because the streaming source needs to read one record at a time, and JSON is the natural format for the file-based simulation of Kafka topics.
- **TSV** (tab-separated): standard output format for Hadoop Streaming MapReduce jobs.
